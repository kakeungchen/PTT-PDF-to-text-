## 为什么做 ptt

市面上的 PDF 转换工具，要么把你的文件传到服务器，要么一遇到扫描件、水印、表格就输出一堆乱码。**ptt** 只坚持一条原则：**宁慢勿错**——并且全程在你的 Mac 上本地完成（OCR 用 macOS 自带的 Vision 框架），文件永远不出设备。

它在最难啃的真实场景上磨出来的：飞书/钉钉导出的超长截图式 PDF（单页高度超过 65000 像素，常规 JPEG 解码器都打不开）、满页防泄密水印、周期性页眉页脚、跨页大表格、嵌入公式。

## 功能

| | |
|---|---|
| **文本型 & 扫描型 PDF** | 文字层直接提取；扫描件和超长截图走本地中英文 OCR |
| **去水印** | 斜向/浅色防伪水印、跨页重复的保密声明自动清除 |
| **去页眉页脚** | 文档编号、页码、logo、周期性重复内容——包括长截图内部的"虚拟分页" |
| **表格还原** | 简单表格保留为 Markdown 表格；跨页大表、合并单元格、多层表头、长说明表格会转成更可读的分组文本 |
| **公式与图示** | OCR 无法忠实线性化的内容（分式、下标、流程图）会标成**公式原文需核对**，绝不伪装成确定正确的公式 |
| **图中文字结构化** | 图示里的文字按版面几何重建成可读表格，人和 AI Agent 都能读懂 |
| **质检自查闭环** | 低置信内容放大 2 倍重新识别；章节、公式、关键数字、指标名会做覆盖审计；仍不确定的内容会**显式标注** |
| **输出干净** | 转换结束自动删除裁切图和中间资产，默认只留下一个 `.md` 文件 |
| **Agent 友好** | 命令行支持 JSON 模式：进度走 stderr，机器可读结果走 stdout |

## 快速开始

### 图形界面

1. 双击 **`启动ptt.command`**（首次运行自动安装依赖，需联网一次约 1–2 分钟，之后完全离线）。
   - 如提示"无法打开"：右键 → 打开 → 再点"打开"。
2. 把 PDF 拖进窗口，勾选输出格式，点 **开始转换**。
3. 结果默认在每个源文件旁的 `转换结果` 文件夹，也可点 **自定义…** 输出到任意位置。

### 命令行 / AI 智能体

```bash
# 基本用法：只输出一个 Markdown 文件
.venv/bin/python -m ptt.cli 文件.pdf -o 输出目录

# Agent 模式：结果 JSON 走 stdout，进度走 stderr
.venv/bin/python -m ptt.cli 文件.pdf --json

# 如需兼容 Word，可显式打开 docx 输出
.venv/bin/python -m ptt.cli 文件.pdf -o 输出目录 -f md docx
```

JSON 包含 `outputs`（生成的文件）、`warnings`（清除了什么水印页眉、自动纠正了什么）、`qa_issues`（需人工复核的位置）、`flagged_blocks`（低置信块数）。

## 工作原理

```
PDF ──▶ 逐页判型（文本层 / 图片）
     ──▶ 文字层提取  /  分片 Vision OCR（1800px 切片 + 重叠去重）
     ──▶ 周期性页眉页脚 / 水印带清除
     ──▶ 表格线网格重建（跨页合并）
     ──▶ 图示 / 公式区域检测 → 内部覆盖账本
     ──▶ 按阅读顺序组装（标题 · 段落 · 表格 · 图片）
     ──▶ 质检：2倍重识别交叉验证 · 词频投票纠错 · 覆盖审计 · 排版审计
     ──▶ Markdown (.md)，可选 Word (.docx)
```

不下载任何模型、不依赖 PyTorch——OCR 是 macOS 系统自带的 Vision 框架，所以整个仓库只有约 40 KB。

## 能力边界（实话实说）

- 烙死在扫描图像里的深色水印无法保证完全清除（浅色和文字型水印效果好）。
- 极小的下标字符（如 K₁）处于 Vision 识别能力边缘——不确定的公式会标注需核对，并阻断“无问题通过”。
- App 截图证据图内部的小字属于尽力识别：不确定区域会显式提示，而不是静默猜测。
- 工具不确定的内容一律显式标注，绝不静默猜测。
